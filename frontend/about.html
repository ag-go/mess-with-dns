About

Why did you build this?

This site exists because I (Julia) was writing a zine about DNS. At the end of many of my zines, I say "the best way to learn is to experiment! Do weird DNS experiments and see how it works". But I realized that most people, even if they have a domain, probably aren't comfortalbe doing weird DNS experiments on a domain that's important to them.

So I figured I'd create a domain that's just for playing around (messwithdns.com) and let anyone set DNS records on it.

This project was built by Julia Evans and Marie Claire LeBlanc Flanagan.

How does it work?

Here are a few notes on how Mess With DNS is implemented, in question/answer form.

How do you parse DNS queries?

It's written in Go and it uses the github.com/miekg/dns library for all if its DNS query parsing and for formatting the replies.

Will my records get deleted?

Yes, records get deleted after a week. This is to heavily discourage anyone from relying on this for anything real. It's just for learning.

Where are the records stored?

They're in a small PostgreSQL database. It contains the records and every DNS response that's it's served in the last 24 hours.

What's the difference between messwithdns.com and messwithdns.com?

The .com version is for serving the website, the .net version is where you're allowed to set DNS records. They're separate because it makes the site faster, more secure, and more reliable (I can use a "real" DNS server for messwithdns.com).

Does it really support setting all DNS record types

Technically the API does allow setting any DNS record type you want, but I've only built the UI for a few types. In particular I haven't set up DNSSEC yet so there's no UI for setting DNSSEC records.

Can I get any subdomain I want?

No, I got worried about phishing so you can only get a randomly generated subdomain, like banana17
